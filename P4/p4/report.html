Problem Definition

        This is a program that will play the game Atropos.  It uses a static evaluator and a minimax search with alpha-beta pruning to determine the move it should make.

Method and Implementation

        The static evaluator uses two scores to calculate the move.  The first one is to see whether a particular position will create traps for the opponent, and additional points if the move will cause the opponent to fall into the trap.  Trap, here, means that the opponent will be forced to take a turn in which he/she will only have one position on the board to color.  It then chooses the color by picking the color that is most seen adjacent to the spot in the “best” position.  The positional score outweighs the highest score for the color, so the two do not interfere with each other.  Positional score is the highest priority.

        The search algorithm uses a minimax search with alpha-beta pruning to look through the tree of possible moves.  It can look five to six moves ahead with reasonable timing.

Experiments

        The program has been tested ten times with a lookahead depth of both five and six, while playing on a board of size seven. Each test was conducted by calculating the win percentage after playing ten games against the default opponent, which plays by choosing a random legal move.

Discussion

        One strength of this method is that the program can look many steps ahead.  The scoring system does not take a very long time, making the program able to look up to six moves ahead with reasonable time (no more than a few seconds).  It also attempts to end the game quickly, rather than filling up the entire board.  It relies on putting the opponent in a position where he will only have an option to choose which color to use.  This means that the program does not make much use of choosing a good color when making it’s move.  It forces the opponent to be able to take only one spot, which is means a good chance of the opponent not being able to make a safe move.  Surprisingly, the lookahead depth of five moves performed worse than the lookahead depth of six moves, although both consistently perform better than random.  This surprising result is likely due to the randomness of the opponent, giving the higher lookahead unlucky numbers and the lower lookahead lucky numbers during testing.

        The scoring system can certainly have a better coloring system.  For the allotted time to complete this project, top priority was given to the position of the move.  The coloring system was tested by trying to color the spot with the color that is both the least and most seen adjacent to the spot.  It seemed that coloring with the color that is most seen adjacent to the spot produced higher winning percentages, so that is the system used in the final version.

Conclusion

        This automatic player can consistently beat the default player more than half the time.  It should be a worthy adversary in the upcoming tournament against other AI!
